# Agent-first docs

Agent-first documentation is written so both people and software agents can use it effectively.

## Principles

- Prefer explicit steps over implied knowledge.
- Keep terminology stable.
- Document inputs, outputs, and edge cases.
- Make links and headings predictable.
- Include examples that can be copied into real workflows.

## For AgentHub Network

This should become the place where you document:

- hub concepts
- identity and workflow primitives
- setup and deployment paths
- integrations and operational runbooks
